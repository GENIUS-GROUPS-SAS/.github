## What is this repository?

This is the **`.github`** special repository for the [GENIUS-GROUPS-SAS](https://github.com/GENIUS-GROUPS-SAS) GitHub organization. It serves as the **single source of truth** for:

- 📄 **Organization profile** — displayed on our GitHub org page
- 🐛 **Issue templates** — standardized forms for bug reports, features, docs, questions, and security
- 🔀 **PR template** — structured pull request template for all repositories
- 📜 **Code of Conduct** — behavioral standards for our community
- 🤝 **Contributing guide** — how to contribute to any GENIUS GROUPS project
- 🛡️ **Security policy** — responsible disclosure and vulnerability reporting
- 📞 **Support guide** — where to get help
- 🏛️ **Governance** — roles, decision processes, RFC and ADR workflows
- 👥 **Maintainers** — who maintains what
- 🏗️ **Architecture standards** — Clean Architecture, DDD, Hexagonal, Event-Driven, API First
- 🛠️ **Development guide** — setup, Docker, CI/CD, testing, coding standards
- 🗺️ **Roadmap** — Vision 2030 across six strategic axes
- 📦 **Release process** — semantic versioning, changelog, release notes
- 💬 **Community guide** — how to get involved
- 💰 **Funding** — sponsorship and enterprise support

## How to Use This Repository

### For GENIUS GROUPS Engineers

When creating a new repository under the GENIUS-GROUPS-SAS organization:

1. **Issue templates** and **PR template** are automatically inherited from this `.github` repository. You don't need to copy them.
2. **CODE_OF_CONDUCT.md**, **SECURITY.md**, **CONTRIBUTING.md**, and **SUPPORT.md** are also inherited if not present in the product repository. We recommend copying them for visibility.
3. **ARCHITECTURE.md**, **DEVELOPMENT.md**, **GOVERNANCE.md**, **RELEASE.md**, and **ROADMAP.md** are organization-level standards. Product repositories may extend them with product-specific details.
4. **LICENSE** — use the proprietary license for closed-source products, and the MIT license for open-source repositories. This file contains both.

### For External Contributors

1. Start with our [Contributing Guide](./CONTRIBUTING.md).
2. Read the [Code of Conduct](./CODE_OF_CONDUCT.md) — we enforce it.
3. Check the [Support Guide](./SUPPORT.md) for the right channel.
4. Review the [Architecture Standards](./ARCHITECTURE.md) to understand how we build.
5. Read the [Development Guide](./DEVELOPMENT.md) to set up your environment.
