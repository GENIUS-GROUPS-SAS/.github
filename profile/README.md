## 🎯 Mission

To design, build, and operate **world-class digital infrastructure** that empowers African businesses, governments, and developers to compete on a global scale — without compromising on local context, regulatory compliance, or user experience.

We believe Africa deserves **first-class technology** — not watered-down imports. Every product we ship is engineered from the ground up for the realities of the markets we serve: mobile-first populations, mobile-money economies, multi-currency environments, low-connectivity zones, and OHADA-compliant legal frameworks.

## 🧬 Our Approach: Product Engineering

We don't separate "product" from "engineering." We practice **Product Engineering** — a discipline where every engineer thinks like a product manager, and every product decision is grounded in engineering reality.

| Principle | What It Means |
|---|---|
| **Product First** | The business need guides every technical decision. We build what users need, not what's technically interesting. |
| **API First** | Every feature is designed as an API before it gets a UI. If it can't be consumed programmatically, it doesn't ship. |
| **Documentation First** | Nothing is merged without documentation. The docs are the contract. |
| **Test First** | Tests accompany or precede implementation. Quality is not a phase — it's a habit. |
| **Security First** | Security is designed in from day one. OWASP Top 10 is the baseline, not the ceiling. |
| **Mobile Money First** | Card payments are an option. Mobile Money is the default. Africa-first means payment-method-first. |
| **Developer Experience** | If a developer can't integrate our API in under 15 minutes, we've failed. DX is a feature. |

## 🏅 Quality Standards

We hold our code to the same bar as the world's best engineering organizations.

| Standard | Requirement |
|---|---|
| **Test Coverage** | ≥ 70% business logic, 100% critical paths |
| **Code Review** | Minimum 1 approval, CI must pass |
| **Static Analysis** | Zero critical issues, zero dead code |
| **Cyclomatic Complexity** | < 10 per function |
| **Function Length** | < 50 lines |
| **File Length** | < 500 lines |
| **Duplication** | < 3% |
| **Accessibility** | WCAG 2.1 AA minimum |
| **Security** | OWASP Top 10 addressed, HMAC verified, TLS 1.3 |
| **API Documentation** | OpenAPI 3.1, interactive Swagger UI |
| **ADR** | Every significant decision documented |
| **CI/CD** | All changes go through CI; production deploys are automated |

## 🏛️ Architecture Principles

| Principle | Description |
|---|---|
| **Clean Architecture** | Dependencies point inward. The domain depends on nothing. |
| **Domain-Driven Design** | Bounded contexts, ubiquitous language, aggregates, domain events. |
| **Hexagonal Architecture** | Ports and adapters. The application core is isolated from infrastructure. |
| **Event-Driven** | Domain events for decoupling. Event sourcing where audit matters. |
| **API First** | OpenAPI spec before code. Mocks before implementation. |
| **Modular Monolith** | Start monolithic, modular by design. Extract services when justified by ADR. |
| **CQRS** | Separate read and write models where scale demands it. |
| **Idempotency** | Every mutating API endpoint supports idempotency keys. |
| **Webhooks** | Signed (HMAC-SHA256), retried with exponential backoff, idempotent. |

## 🔓 Open Source Philosophy

> **Open by default, proprietary by exception.**

- **Infrastructure libraries**, **SDKs**, **templates**, and **tooling** are open source.
- **Core business products** (GeniusPay, Genius Invoice, etc.) are proprietary.
- Every open-source repository ships with: `LICENSE`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `README.md`.
- We accept external contributions. See [`CONTRIBUTING.md`](../CONTRIBUTING.md).
- All open-source projects follow Semantic Versioning and maintain a `CHANGELOG.md`.
- We sponsor maintainers and contribute upstream to the libraries we depend on.
